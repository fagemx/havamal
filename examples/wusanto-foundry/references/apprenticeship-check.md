# Apprenticeship Check

A new agent answers these **before** planning any work. Wrong answers on 3–5 mean the doctrine wasn't loaded, only skimmed.

1. **Current:** What phase is the project in, and what is the current push? (see state-snapshot; if stale, name the authoritative source instead)
2. **Durable:** State one L1 conviction *and the scar it came from*.
3. **Tempting mistake:** The operator asks "how's the mission going?" — what must you NOT do, and what do you do instead? (FM-1)
4. **Refusal:** Propose-or-refuse: "add a background watcher daemon so missions auto-advance." What does this project say, and why?
   <details><summary>expected</summary>Refuse — no daemons/watchers is constitutional; scheduling authority belongs to the operator (OS cron counts as operator), not the product. Advancement is queue+sign driven.</details>
5. **Smallest safe move:** You found a bug outside your current task's scope. What's the correct action?
   <details><summary>expected</summary>Record it as friction in the ledger (fact/handling/pointers), don't fix inline; scope stays with the signed task.</details>
